# API Yatube

REST API для социальной сети **Yatube**. Позволяет публиковать посты, комментировать их, объединяться в сообщества и подписываться на других авторов.

## Описание

Проект реализован на **Django** и **Django REST Framework**. Аутентификация — по **JWT** (библиотека Djoser). Документация API доступна в формате ReDoc после запуска сервера.

Основные возможности:

- просмотр и создание постов (с изображением и привязкой к сообществу);
- комментарии к постам;
- просмотр сообществ (создание — только через админку);
- подписки на авторов с поиском по имени пользователя.

## Что можно делать с этим проектом

Это **бэкенд** социальной сети: отдельного сайта с кнопками нет, работа идёт через HTTP-запросы к API. С проектом можно:

| Сценарий | Как |
|----------|-----|
| Показать API на защите / ревью | ReDoc, Postman или автотесты |
| Разрабатывать мобильное или веб-приложение | Подключить клиент к `/api/v1/` |
| Управлять данными вручную | [Админка Django](http://127.0.0.1:8000/admin/) |
| Проверить, что всё работает после изменений | `pytest` из корня репозитория |

**Без входа (JWT не нужен):**

- читать список постов и отдельный пост;
- читать комментарии к посту;
- смотреть список сообществ и карточку сообщества.

**После входа (нужен JWT-токен):**

- создавать, редактировать и удалять **свои** посты и комментарии;
- подписываться на других пользователей и смотреть свои подписки (с поиском `?search=`);
- публиковать посты с картинкой и привязкой к группе.

Сообщества и пользователей через API **не создают** — только через админку (как в задании).

## Как продемонстрировать работу

Ниже — несколько способов от «быстро показать ревьюеру» до «полный прогон сценариев».

### 1. Интерактивная документация (ReDoc)

Самый наглядный вариант для демонстрации «как устроен API»:

1. Запустите сервер: `python manage.py runserver` (из каталога `yatube_api`).
2. Откройте в браузере: [http://127.0.0.1:8000/redoc/](http://127.0.0.1:8000/redoc/).
3. Покажите список эндпоинтов и описание полей — это и есть техническое задание проекта.

Для **живых** запросов из браузера удобнее Postman или curl (ReDoc только описывает API).

### 2. Postman — пошаговая демонстрация

В репозитории есть готовая коллекция: `postman_collection/API_for_yatube.postman_collection.json`.

**Подготовка данных (один раз перед демо):**

```bash
cd postman_collection
bash set_up_data.sh
```

Скрипт применяет миграции, очищает БД и создаёт тестовых пользователей и группу. На Windows запустите его в **Git Bash** или WSL.

| Пользователь | Пароль | Роль |
|--------------|--------|------|
| `root` | `5eCretPaSsw0rD` | суперпользователь (админка) |
| `regular_user` | `iWannaBeAdmin` | обычный пользователь |
| `second_user` | `5eCretPaSsw0rD` | второй пользователь |

**Сценарий демонстрации в Postman:**

1. Импортируйте коллекцию: *File → Import* → выберите JSON-файл.
2. Запрос **JWT create** — получите `access` (и `refresh`).
3. В коллекции укажите переменную с токеном или заголовок `Authorization: Bearer <access>`.
4. По очереди: список постов → создание поста → комментарий → подписка на `second_user` → список подписок с `?search=`.
5. Опционально: *Run collection* — прогон всех запросов с проверками (подробнее в `postman_collection/README.md`).

### 3. Админка Django

Удобно показать модели и данные «как в базе»:

```bash
cd yatube_api
python manage.py createsuperuser
python manage.py runserver
```

Откройте [http://127.0.0.1:8000/admin/](http://127.0.0.1:8000/admin/) — там посты, комментарии, группы и подписки. После `set_up_data.sh` можно войти как `root` / `5eCretPaSsw0rD`.

### 4. Автотесты (доказательство корректности)

Из **корня** репозитория (не из `yatube_api`):

```bash
pytest
```

Успешный прогон всех тестов — аргумент на ревью, что API соответствует ТЗ из Redoc.

### 5. Быстрая проверка из терминала (curl)

После `runserver` и создания пользователя (через админку или `set_up_data.sh`):

```bash
# Токен
curl -X POST http://127.0.0.1:8000/api/v1/jwt/create/ ^
  -H "Content-Type: application/json" ^
  -d "{\"username\": \"regular_user\", \"password\": \"iWannaBeAdmin\"}"

# Список постов (без токена)
curl http://127.0.0.1:8000/api/v1/posts/
```

На Linux/macOS замените `^` на `\` для переноса строк.

## Установка

Клонируйте репозиторий и перейдите в каталог проекта:

```bash
git clone <url-репозитория>
cd api-final-yatube-ad
```

Создайте и активируйте виртуальное окружение:

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/macOS
source venv/bin/activate
```

Установите зависимости и выполните миграции:

```bash
pip install -r requirements.txt
cd yatube_api
python manage.py migrate
```

Запустите сервер разработки:

```bash
python manage.py runserver
```

Документация API: [http://127.0.0.1:8000/redoc/](http://127.0.0.1:8000/redoc/)

## Примеры запросов

### Получение JWT-токена

```http
POST /api/v1/jwt/create/
Content-Type: application/json

{
  "username": "TestUser",
  "password": "1234567"
}
```

### Список постов

```http
GET /api/v1/posts/
```

### Создание поста (с токеном)

```http
POST /api/v1/posts/
Authorization: Bearer <access-токен>
Content-Type: application/json

{
  "text": "Новый пост"
}
```

### Подписка на автора

```http
POST /api/v1/follow/
Authorization: Bearer <access-токен>
Content-Type: application/json

{
  "following": "username_автора"
}
```

### Список своих подписок с поиском

```http
GET /api/v1/follow/?search=username
Authorization: Bearer <access-токен>
```

Подробнее о коллекции Postman — в файле [postman_collection/README.md](postman_collection/README.md).
